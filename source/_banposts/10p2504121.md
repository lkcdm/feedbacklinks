## 环境需求

首先，根据[Hexo](https://hexo.io/zh-cn/)官方要求，需要`Git`（推送的）`Node.js`（必要的）。

![image-20250412164118404](https://cn-sy1.rains3.com/koimg/2025/04/286e00882acda137e819990682841625.png)

当然，这一般来说，对某些盆友来说过于“复杂”，所以祭出我们的老祖《[宝塔面板 - 简单好用的Linux/Windows服务器运维管理面板](https://www.bt.cn/new/index.html)》了，在宝塔进行操作可视化更简单一点。

> PS：此处先写的是在服务器上进行操作，不包含推送Github，服务器玩家使用方案，如果是本地+推送Github、vercel部署的也大差不差，满足环境就行。

首先，先去宝塔面板软件商店里头下一个`Node.js版本管理器`。

![image-20250412165018922](https://cn-sy1.rains3.com/koimg/2025/04/738d0605ad1956c021532d1eff0a4909.png)

## 操作正文

然后开始正式操作，创建网站和目录就不多说了，直接开始配置，先搞一个皇帝的目录，打开终端下载hexo。

![image-20250412170150160](https://cn-sy1.rains3.com/koimg/2025/04/cc131a9503c1a46359e1d2a55d710083.png)

```
npm install -g hexo-cli
```

![image-20250412170241382](https://cn-sy1.rains3.com/koimg/2025/04/ba0e6b93719fa56724c3c89d2795df89.png)

```
hexo init 目录
```

![image-20250412180908291](https://cn-sy1.rains3.com/koimg/2025/04/a84ab685ea41a06d82a7384dbbb74624.png)

### 框架结构

```
.
├── _config.yml 网站的配置文件
├── package.json 应用程序的信息
├── scaffolds 模版文件夹
├── source 资源文件夹
|   └── _posts 文章
└── themes 主题文件夹
```

![image-20250412181249528](https://cn-sy1.rains3.com/koimg/2025/04/eecce2be8f6ea2f1ea5c76fd63357e8d.png)

### 构建博客

完事之后，然后就可以执行操作，一共有两种方式：

- Linux 用户可以将 Hexo 所在的目录下的 `node_modules` 添加到环境变量之中即可直接使用 `hexo <command>`：

```
echo 'PATH="$PATH:./node_modules/.bin"' >> ~/.profile
```

- `npx hexo <command>`直接执行。

构建命令：`hexo g`或者`npx hexo g`。

![image-20250412181553750](https://cn-sy1.rains3.com/koimg/2025/04/7b64a89045a5251f086d47f34adbf6c5.png)

这时候在刷新一下目录就可以看到他生成了一个`public`文件夹，这就是构建好的静态博客本体了。

![image-20250412181622263](https://cn-sy1.rains3.com/koimg/2025/04/81b718c65136a8d7825e017bbd0912c2.png)

![image-20250412181703909](https://cn-sy1.rains3.com/koimg/2025/04/288a7d04a06193fb2f8ffdfd8ae06b66.png)

然后去网站设置，将允许目录修改成构建静态文件的文件夹即可，解析后就可以正常访问到博客了。

![image-20250412182132876](https://cn-sy1.rains3.com/koimg/2025/04/651da20a2d6753e6dc1bb221445472e5.png)

![image-20250412182210672](https://cn-sy1.rains3.com/koimg/2025/04/24f487fba6e5652c13e8c18ea772ac2f.png)

1. 使用 NPM 包进行安装

   ```
   npm i hexo-theme-solitude
   ```

   ![image-20250412182636065](https://cn-sy1.rains3.com/koimg/2025/04/f8d000ef6c172e6ef84bae43b3e76f3b.png)

2. 去配置文件里头修改代码，应用主题

   ```
   theme: solitude
   ```

![image-20250412182710498](https://cn-sy1.rains3.com/koimg/2025/04/70ac07b9104b6c05f851ed022908576c.png)

然后再使用`hexo g` `hexo d`进行构建更新

> hexo d 用于清理缓存，建议修改主题的话都用一下，就是我们熟知hexo的“hexo三连（hexo d|s|g）

![image-20250412182735661](https://cn-sy1.rains3.com/koimg/2025/04/9cdbc81504b71d2f2808af7fbdcfa23c.png)

**建议的话还是去看一眼主题文档吧：[快速开始 | Solitude 文档](https://solitude.js.org/guide/quick-start.html#主题安装)**

## vercel部署方案

如果你完成了以上的操作并且顺利的话，那么，恭喜你，超过了99%的开发者们（bushi）。<AFace face="emoticon:8"/>

那么，有人就问了，主播主播，服务器太贵了没法一直续费怎么办？

唉！你问对了，还真有，除了github，我们还可以用vercel，vercel的优点就是速度不错，可绑定自己的域名，并且还有加速CNAME。

### 推送操作

[klcdm/GitAutumnPushtutorial: GitAutumn 简单推送使用教程 - GitAutumnPushtutorial - GitAutumn](https://gitea.koxiuqiu.cc/klcdm/GitAutumnPushtutorial)

大差不差，只不过是私人仓库，照搬就行；写了无数次重复的桥段是真的会疯掉的。<AFace face="emoticon:9"/>

### 部署vercel

前往[Login – Vercel](https://vercel.com/login)，注册账户，使用Github登录就行了。

![image-20250412184835264](https://cn-sy1.rains3.com/koimg/2025/04/691b71933e29d70dfe1905daf638ac39.png)

然后创建项目。

![image-20250412184947412](https://cn-sy1.rains3.com/koimg/2025/04/8ad838d981fc4043b071da1f84df1a82.png)

弹出选项后，我们选择Github就行，这样就可以看到你的Github项目了。

![image-20250412185015242](https://cn-sy1.rains3.com/koimg/2025/04/ad5303c4ab660c80f2c47d46878691df.png)

如果没有看到你的博客项目，可点你进行添加就行。

![image-20250412185126180](https://cn-sy1.rains3.com/koimg/2025/04/3f98ff96b9c382af405cba5da1d659d5.png)

其余设置可以不用动，`Framework Preset`我们选择`Other`然后直接`Deploy`。

![image-20250412185212918](https://cn-sy1.rains3.com/koimg/2025/04/9fca2bede84e1c64cfb82b796c299cd0.png)

完成后，点进此项目管理面板，找到`domains`绑定域名。

![image-20250412185403978](https://cn-sy1.rains3.com/koimg/2025/04/ce13d08c38b8cf775e0d4fb122e48cd9.png)

![image-20250412185455533](https://cn-sy1.rains3.com/koimg/2025/04/a846034bdf8c3860ed54f3d1cb5c530a.png)

绑定完成之后，查看，它会给你一个解析值解析即可。

![image-20250412185530336](https://cn-sy1.rains3.com/koimg/2025/04/2fc81adf3ff998bdc14f0250daa208e7.png)
